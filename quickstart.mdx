## Run your first scan

<Steps>
  <Step title="Install Gitleaks">
    Install Gitleaks using your preferred method. On macOS with Homebrew:

    ```bash
    brew install gitleaks
    ```

    For other platforms, see the [installation guide](/installation).
  </Step>

  <Step title="Scan a git repository">
    Run `gitleaks git` against any local git repository. Use `-v` to print each finding to the terminal as it is found:

    ```bash
    gitleaks git -v /path/to/your/repo
    ```

    If you run the command from inside the repository directory, you can omit the path:

    ```bash
    cd /path/to/your/repo
    gitleaks git -v
    ```

    The `git` command scans the full commit history using `git log -p` under the hood.
  </Step>

  <Step title="Understand the output">
    When Gitleaks finds a secret, it prints a structured finding. Each field identifies what was found, where it lives, and who introduced it:

    ```text
    Finding:     "export BUNDLE_ENTERPRISE__CONTRIBSYS__COM=cafebabe:deadbeef",
    Secret:      cafebabe:deadbeef
    RuleID:      sidekiq-secret
    Entropy:     2.609850
    File:        cmd/generate/config/rules/sidekiq.go
    Line:        23
    Commit:      cd5226711335c68be1e720b318b7bc3135a30eb2
    Author:      John
    Email:       john@users.noreply.github.com
    Date:        2022-08-03T12:31:40Z
    Fingerprint: cd5226711335c68be1e720b318b7bc3135a30eb2:cmd/generate/config/rules/sidekiq.go:sidekiq-secret:23
    ```

    | Field | Description |
    |-------|-------------|
    | `Finding` | The full line or string that matched the rule |
    | `Secret` | The extracted secret value |
    | `RuleID` | The ID of the rule that triggered |
    | `Entropy` | Shannon entropy of the secret — higher values indicate more randomness |
    | `File` | Path to the file containing the secret |
    | `Line` | Line number within the file |
    | `Commit` | Git commit SHA where the secret was introduced |
    | `Author` / `Email` | The committer who introduced the secret |
    | `Fingerprint` | Unique identifier for this specific finding |

    At the end of the scan, Gitleaks prints a summary of how many bytes were scanned, how long it took, and how many leaks were found.
  </Step>

  <Step title="Export a report">
    Save findings to a file using `--report-path`. The format is inferred from the file extension, or you can set it explicitly with `--report-format`:

    ```bash
    # Save as JSON (format inferred from extension)
    gitleaks git --report-path gitleaks-report.json

    # Save as SARIF for IDE or GitHub Code Scanning integration
    gitleaks git --report-path report.sarif --report-format sarif

    # Save as CSV
    gitleaks git --report-path report.csv --report-format csv
    ```

    Supported formats are `json`, `csv`, `junit`, `sarif`, and `template`.
  </Step>
</Steps>

## Common flags for first-time users

| Flag | Description |
|------|-------------|
| `-v`, `--verbose` | Print each finding to the terminal as it is found |
| `-r`, `--report-path` | Path to write the report file |
| `-f`, `--report-format` | Output format: `json`, `csv`, `junit`, `sarif`, `template` |
| `-c`, `--config` | Path to a custom `.gitleaks.toml` configuration file |
| `--exit-code` | Override the exit code when leaks are found (default: `1`) |
| `--baseline-path` | Path to a previous report — only new findings are reported |

<Tip>
Use `--baseline-path` when scanning a repository with an existing history of known findings. Generate a baseline first with `gitleaks git --report-path baseline.json`, then pass `--baseline-path baseline.json` on subsequent runs to surface only new issues.
</Tip>

## Exit codes

Gitleaks exits with a predictable code so it integrates cleanly into CI scripts:

```text
0   - no leaks present
1   - leaks or error encountered
126 - unknown flag
```
