Gitleaks is an open-source SAST tool that scans git repositories, directories, and arbitrary input for hardcoded secrets. With 150+ built-in detection rules and a flexible TOML configuration system, Gitleaks helps teams prevent credentials from leaking into source code.

## How it works

Gitleaks scans content using detection rules — each rule contains a regular expression, optional entropy threshold, and keyword filters. When a match is found, Gitleaks reports it as a finding with full context: the matched secret, file, line, commit, author, and a unique fingerprint.

<Steps>
  <Step title="Install Gitleaks">
    Download and install the binary for your platform. Available via Homebrew, Docker, or direct binary download.
  </Step>
  <Step title="Run a scan">
    Point Gitleaks at a git repository, directory, or pipe data through stdin. Gitleaks uses its built-in ruleset by default.
  </Step>
  <Step title="Review findings">
    Findings are printed to the terminal with full context. Export to JSON, CSV, JUnit, or SARIF for integration with other tooling.
  </Step>
  <Step title="Tune with configuration">
    Add a `.gitleaks.toml` to your repository to extend rules, add allowlists, and suppress false positives.
  </Step>
</Steps>
